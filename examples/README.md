# Examples

These examples show how HADF templates may be used in practice.

They are intentionally compact.

The goal is to demonstrate the shape of useful specification and review context, not to prescribe exact wording for every repository.

## Included Examples

- `small-change-example.md` - a narrow, reversible copy change
- `feature-plan-example.md` - a decision-complete plan for a material behaviour change
- `delivered-spec-example.md` - the delivered specification for that feature plan
- `messy-ambiguity-example.md` - a request that should stop for clarification before implementation

The examples use a fictional task management application so the workflow can be understood without repository-specific knowledge.
