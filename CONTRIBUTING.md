# Contributing

Feedback and improvements are welcome.

This repository is a framework and template set, so contributions should preserve clarity, concision, and practical usefulness.

## Good Contributions

- Clarify language without changing meaning
- Identify duplication, ambiguity, or adoption friction
- Improve templates while preserving their purpose
- Add concise worked examples
- Surface contradictions between framework sections
- Suggest missing guidance that reduces implementation loops

## Contributions To Avoid

- Turning the framework into a general essay about AI
- Adding speculative ideas without an operational purpose
- Expanding process for its own sake
- Adding tool-specific guidance to the core framework when it belongs in examples or templates

## Issues

Use issues for:

- Questions about interpretation
- Adoption friction
- Proposed framework changes
- Template improvements
- Example requests

## Pull Requests

Pull requests should explain:

- What problem the change solves
- Whether it changes the framework, templates, examples, or repository packaging
- Why the change belongs in this repository

Framework changes should stay aligned with the core purpose: reliable human-agent delivery with fewer unnecessary implementation loops.
